## \* 오늘 + 내가 한 일

- 전 날에 지점의 지역적 특성을 봤다면
- 오늘은 데이터에 있는 고객들의 특성을 분석
- 체험 일자별로 1, 2, 3 일차로 분리 후 각 체류시간, 체험까지의 오는 시간, 결제율 등을 확인
- 공유 오피스 서비스를 단순히 많이 이용한다해서, 결제로 이어지지 않는다는 음의 방향성 다시 한번 확인
- 분석한 인사이트들을 반영하여 비즈니스 전략 아이디어 논의
  - 회원권 추가 생성 : 체류시간 음의 방향성, 3일체험후 이탈 유저(가격설득)
  - 첫 경험 개선 방향 : 하루 이용한 유저를 설득해서 → trial_day를 늘리게 만들고 → 만든 회원권에서 가격설득을 통해 결제유도
  - 사무직 집중 지역 고려하여 입지 선정 등

## \* 사용했던 코딩

- trial_day == 1 내에서
- trial_gap(신청 후 첫 방문까지의 일수갭)0 ~ 1/ 2 ~ 3 그룹인 유저의 평균 체류 시간 확인

```python
# 각 그룹 평균 체류시간 계산
avg_stay_01 = group_01["stay_time_second_total"].mean()
avg_stay_23 = group_23["stay_time_second_total"].mean()

print("0~1 그룹 평균 체류시간:", avg_stay_01)
print("2~3 그룹 평균 체류시간:", avg_stay_23)

# 시각화
plt.figure(figsize=(6,5))
plt.bar(["0~1 그룹", "2~3 그룹"],
        [avg_stay_01, avg_stay_23],
        color=["steelblue", "darkorange"])
plt.title("1일차 유저 Trial Gap 그룹별 평균 체류시간")
plt.ylabel("평균 체류시간 (초)")
plt.show()
```

## \* 회고

- 체험 데이터는 단순히 많이 쓰는 고객이 결제로 이어진다는 가설을 깨고, **이용 패턴(방문 시점, Gap 등)**이 전환에 더 중요한 요인임을 다시 한번 확인함
