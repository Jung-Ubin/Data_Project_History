### 내가 생각해본, 아이디어 정리(정리 안된 ver)(이렇게 아이템을 만들어보면 어떨까?)

1. 현 앱 시스템 개편 및 일일 퀘스트 생성

- 질문 1회 완료 시 → 초성 무료 힌트 1회 제공

- 친구 힌트피스 보내기 → 보낼 때 n개 획득 받을 때 n<개 획득

  - n개 수집 시 온전한 초성(or 다른 아이템 변경가능)힌트 1회권 획득
  - n명 초대 시 아이템 제공(칭호, 하트포인트, 초성힌트권 등)

- 채팅방

  - 질문 초성 추측 후 본명 맟추기 시스템으로 변경
  - 친구 이름 정답 시 → 무료 채팅 활성화(or 다른 아이템으로 변경 가능)

- 레벨 시스템 도입(프로필 관련)
  - 이름을 맟출 때 마다, 경험치 획득
  - 랭킹 시스템 / 칭호 시스템 추가 → 유저들에게 성취감 강화

2. 질문 탐색만 반복하는 유저

- 질문 타겟 연결이 안맞는다고 보여짐 → "질문을 완료할 이유 없다고 생각됨"
  - 학교 단위 인기 질문 피드 제공 등

3. 알림 센터 콘텐츠 강화

   - 누군가 사용자를 투표 시 멘트 수정 " 누군가 oo님이 ~ 에 뽑았어요!"
   - 기간제 아이템들을 만들어서 "oo아이템이 오늘 만료되요" 등
   - 사용자 학교 유저 조사하여 멘트 알람으로 학교유행질문 홍보

4. 결제 전환

- 학생층은 구매 → 게임
  - 아이템을 생성 → 이름 공개권 등 생각중

### 팀원과 의논 후 정리해본 개선해야될 문제점(ver.01)

< 서비스 문제점에 관한 개선방안 아이디어 >

1. 단발성 서비스

- 지역별로 유행하는 패턴확인 -> 아직 유행이 오지 않은 지역 중심으로 활성화
- 밈 위주의 질문으로 유행을 빠르게 반영하여 지속적 흥미 유발

2. 질문 탐색하면서 이탈 높음

- 앱 시작 시 관심있는 주제 선택하게하여 유저에게 최적의 질문 제공 알고리즘 생성
- 질문 생성 채택 이벤트 : 질문 생성한 유저들 중 채택된 질문 배달 기프티콘 제공  
   -> 유저들의 질문 생성 활동을 활성화 시키고 수집된 질문 데이터로 어떤 주제에 흥미가 있는지 분석 -> 질문 개선에 도움

3. 결제컨텐츠 부족

- 게시판 개설 (대나무숲) : 인근 학교와 소통할 수 있는 커뮤니티 생성  
   -> 더 많은 학교, 학생에게 게시글을 보이게 하기 위해 하트 사용 (확성기 기능)

- 질문과 관련된 광고 도입 : 전시에 관한 광고료를 받았다고 가정. 같이 전시회 가고싶은 친구는? 이란 질문에 투표를 하고 상대방에게도 반응이 오는 경우 전시 관련된 광고 팝업을 띄우거나 할인 쿠폰을 제공.

## \* 사용했던 코드

### 정제된 유저들의 User_flow 공유용 테이블 생성

```sql
CREATE TABLE final.session_count_1_log_e AS
SELECT
     dus.num,              -- 기존 PK (순번)
     dus.device_id,
     dus.user_id,
     dus.osname,
     dus.session_id,
     dus.session_count,
     scl.event_flow        -- 새로 추가되는 event_flow 컬럼
 FROM final.device_user_id_summary AS dus
 LEFT JOIN final.session_count_1_log AS scl
     ON dus.session_id = scl.session_id
 ORDER BY dus.user_id;
```
