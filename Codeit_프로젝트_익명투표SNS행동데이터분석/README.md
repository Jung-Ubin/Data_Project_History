# 익명 투표 SNS 행동 데이터 분석

Codeit 데이터 분석 프로젝트 (팀 프로젝트)  
프로젝트 기간: 25.10.10 ~ 25.11.14

> 약 19만 명의 유저 행동 로그 데이터를 기반으로   
> 익명 투표 SNS 서비스의 **단발성 이용 문제와 낮은 재방문율의 원인을 분석**하고,  
> 사용자 경험 **개선 및 리텐션 강화를 위한 데이터 기반 개선안을 제안**한 프로젝트입니다.

---

## 프로젝트 개요
본 프로젝트는 익명 투표 기반 SNS 서비스의 사용자 행동 로그를 분석하여,   
앱 사용 후 재방문 하지 않는 단발성 유저 비율이 높은 구조적 원인을 파악하고  
서비스 활성화를 위한 실행 가능한 전략을 도출하는 것을 목표로 진행되었습니다.

분석 결과, 전체 유저 중 약 45%가 앱을 1회만 사용 후 이탈하는 것으로 나타났으며,   
이를 해결하기 위해 유저 행동 흐름(User Flow)과 투표 주제 콘텐츠 구조를 중심으로 분석을 수행했습니다.

---

## 분석 목표
- 유저 행동 로그 기반 앱 사용 흐름(User Flow) 분석
- 단발성 유저와 재방문 유저의 행동 패턴 차이 도출
- 투표 주제 콘텐츠의 구조적 문제 및 이탈 지점 파악
- 리텐션 및 참여도 강화를 위한 UX·서비스 개선안 제안

---

## 사용 데이터
- 유저 행동 로그 데이터(약 19만 명)
- 투표 주제 데이터
- 신고 데이터(신고 사유, 신고 횟수)

--- 

## 분석 방법
- 유저 이벤트 로그 전처리 진행
- User Flow 분석
  - 이벤트 로그 시간 순 정렬  
  - 세션(앱 시작 이후 종료까지의 이벤트 로그 묶음) 이상치 90분 기준 단위로 분리

- PrefixSpan 기준 순차 패턴 분석
  - 유저들의 앱 내에서의 행동패턴 추출

- 투표 콘텐츠 분석
  - 주제 분류
  - 주제별 신고 횟수 비교

---

## 주요 분석 결과 및 인사이트

### 1. 유저 행동 흐름 분석
- 유저 행동의 대부분이 질문 탐색 중심 플로우에 집중
- 투표 주제를 반복 탐색만 하다 종료하거나, 동일 행동을 반복하는 패턴 다수 확인
- **투표 완료 이벤트가 없는 유저가 약 80%** 로 가장 높은 비중 차지  
  → 앱의 핵심 경험(투표 완료)에 도달하지 못하고 이탈하는 구조적 문제 확인

### 2. 단발성 유저 비율 분석
- 전체 유저 중 약 45%가 1회 사용 후 이탈
- 신규 유저 유입 비율은 약 2% 수준으로 매우 낮음  
- 서비스 구조상 앱 내 친구가 다수 있어야 흥미를 느끼는 진입 장벽 존재

### 3. 질문 콘텐츠 분석
- 질문 주제 중 '연애/호감' 카테고리의 신고 빈도가 통계적으로 유의하게 높음
- 신고 사유 대부분이 "그냥 싫다, 불쾌한 내용이다" 로 나타남  
  → 투표 주제 품질과 개인 취향을 반영하지 못하는 구조가 이탈 원인으로 작용

### 4. 결제 및 상점 흐름 분석
- 상점 접근률 및 결제 전환율 모두 매우 낮은 수준
- 결제 아이템의 사용 목적이 명확하지 않아 구매 가치 인식 부족

---

## 개선안 제안

### 투표 행동 이탈률 개선
- 회원 가입 시 선호/기피 질문 주제 선택 기능 도입
- 투표 1세트(10건) 완료 시 보상형 하트 포잍느 시스템 제안
- AI 기반 투표 주제 생성 도우미 도입으로 투표 주제 생성 진입 장벽 완화

### 단발성 유저 재활성화
- 투표 외에도 참여 가능한 인근 학교 단위 **익명 소통 창구 기능** 제안
- 친구 수와 무관하게 이용 가능한 공간 제공으로 진입 장벽 완화

### 결제 아이템 가치 재정의
- 기존 투표자를 추리할 때 사용되는 초성 힌트 기반 결제 구조 개선
- '이름 맞추기' 기능 도입으로 결제 아이템 사용 목적 명확화

### 사용자 온보딩 개선
- 친구 추가 이전에도 앱 핵심 기능을 체험할 수 있도록 튜토리얼 구조 개선 제안
